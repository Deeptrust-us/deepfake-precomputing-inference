# Audio Quad-Stream Deepfake Detection

A PyTorch implementation of an **audio** quad-stream deepfake detector that classifies audio as **real** or **fake** using four 1×224×224 time-frequency “images” per sample:

- segment STFT
- segment Log-Mel
- full-audio STFT
- full-audio Log-Mel

Each training/evaluation sample uses **one selected speech segment** (via VAD) paired with the full-audio features for the same WAV.

## Key idea

For each WAV file, preprocessing:

- runs VAD and selects a speech segment (`seg_num = 0, 1, 2, ...`)
- saves **one** segment feature pair as `<stem>_<seg_num>.npy` (recommended: `--segment-mode random_one`)
- saves full-audio features once as `<stem>.npy`

Training loads **(segment_stft, segment_logmel, full_stft, full_logmel)** as the 4 model inputs.

## Quick start (commands)

```bash
source venv/bin/active

# 1) Preprocess features
# Recommended: write EXACTLY ONE segment per WAV (training/eval expects this)
python scripts/preprocess_audio.py --dataset-root /path/to/dataset_audios --segment-mode random_one --seed 42
# Disk-saving option: delete source WAVs after feature extraction (irreversible)
python scripts/preprocess_audio.py --dataset-root /path/to/dataset_audios --segment-mode random_one --seed 42 --delete-audio

# 2) Train
# Training reads the dataset path from config/config.yaml (data.data_root)
python scripts/train.py --config config/config.yaml

# 3) Evaluate
python scripts/evaluate.py --config config/config.yaml --checkpoint checkpoints/best_model.pth --split test
```

## Dataset layout

Required:

```
dataset_audios/
├── labels.json
└── audio/
    ├── 0000000001.wav              # Option A (preferred)
    └── real/0000000001.wav         # Option B (fallback)
       fake/0000001234.wav
```

Generated by preprocessing:

```
dataset_audios/
└── features/
    ├── segment_stft/0000000001_0.npy        # exactly one segment file per stem (recommended)
    ├── segment_logmel/0000000001_0.npy
    ├── full_stft/0000000001.npy
    └── full_logmel/0000000001.npy
```

## Project structure (relevant parts)

```
deepfake_audio/
├── scripts/
│   ├── preprocess_audio.py     # WAV → VAD segment selection → 4-stream feature .npy files
│   ├── train.py                # Training loop
│   └── evaluate.py             # Evaluation
└── src/
    ├── data/dataset.py         # DeepfakeDataset (one sample per labels.json entry)
    ├── models/quad_stream.py   # Audio quad-stream model (kept name QuadStreamModel)
    └── utils/audio_features.py # STFT/log-mel/VAD helpers
```
